# Anatomime Shared Room Game Run Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace Anatomime's single shared-session model with a reusable room plus game-run model that supports Host Judged, Device Typed, and Device Multiple Choice team play under one join code.

**Architecture:** Keep the public Anatomime routes and join URLs stable while moving shared gameplay internals to new room/run Prisma models and a focused server service. Put deterministic gameplay decisions in pure helpers first, then wire Prisma transactions, API routes, host UI, player UI, realtime refresh, and browser tests around those helpers.

**Tech Stack:** Next.js App Router, React client components, Prisma/Postgres, existing anatomy study helpers, existing flashcard progress helper, node:test, Playwright.

export type AnatomimeGameRunPhase = "LOBBY" | "ACTIVE_TERM" | "TURN_REVIEW" | "GAME_COMPLETE"
export type AnatomimeTermOutcome = "pending" | "got" | "missed" | "stolen"
export type AnatomimeAnswerKind = "typed" | "choice"
export type AnatomimeGuessFeedbackKind =

  | "active-correct"
  | "opposing-correct-held"
  | "opposing-team-already-held"
  | "practice-correct"
  | "timeout-missed"
  | "timeout-steal-awarded"
  | "host-judged-correct"
  | "locked"

export const ANATOMIME_TERM_SECONDS = 30
export const ANATOMIME_TYPED_GUESS_LIMIT = 5
export const ANATOMIME_CHOICE_GUESS_LIMIT = 1
export const ANATOMIME_ROOM_IDLE_MINUTES = 30
export const ANATOMIME_REVIEW_WINDOW_MINUTES = 30
export const ANATOMIME_HOST_IDLE_SECONDS = 60
export const ANATOMIME_ELECTION_SECONDS = 60

export type AnatomimeTermCredit = {
  playerId: string
  teamId: string
  userId: string | null
  submittedAt: Date
}

export type AnatomimeTermState = {
  cardId: string
  activeTeamId: string
  outcome: AnatomimeTermOutcome
  firstActiveCorrect: AnatomimeTermCredit | null
  firstOpposingCorrect: AnatomimeTermCredit | null
}

export function createInitialTermState(input: { activeTeamId: string; cardId: string }): AnatomimeTermState {
  return {
    cardId: input.cardId,
    activeTeamId: input.activeTeamId,
    outcome: "pending",
    firstActiveCorrect: null,
    firstOpposingCorrect: null,
  }
}

export function typedGuessStatus(priorTypedAttempts: number) {
  const allowed = priorTypedAttempts < ANATOMIME_TYPED_GUESS_LIMIT
  return {
    allowed,
    remainingAfterUse: allowed ? Math.max(ANATOMIME_TYPED_GUESS_LIMIT - priorTypedAttempts - 1, 0) : 0,
  }
}

export function choiceGuessStatus(priorChoiceAttempts: number) {
  const allowed = priorChoiceAttempts < ANATOMIME_CHOICE_GUESS_LIMIT
  return {
    allowed,
    remainingAfterUse: allowed ? 0 : 0,
  }
}

export function calculateMultipleChoiceUnlockSeconds(labels: string[]) {
  const totalChoiceChars = labels
    .slice(0, 4)
    .reduce((sum, label) => sum + label.trim().length, 0)
  return Math.min(10, Math.max(5, 5 + Math.ceil(Math.max(0, totalChoiceChars - 80) / 30)))
}

Implement the functions with these constraints:

- Active-team correct guesses set `firstActiveCorrect`, return `scoreTeamId` as the active team, and set `shouldAdvance` to `true`.
- First opposing correct guess sets `firstOpposingCorrect`, returns no score, and keeps `shouldAdvance` false.
- Later opposing correct guesses return `practice-correct` or `opposing-team-already-held` with no score and no progress credit.
- Incorrect guesses leave state unchanged and return `incorrect`.
- Timeout with a held opposing correct answer returns score for that opposing team and outcome `stolen`.
- Timeout without a held opposing correct answer returns no score and outcome `missed`.

Add room and election helpers:

```ts
export function canJoinRoom(
  room: {
    status: string
    endedAt: Date | null
    reviewExpiresAt: Date | null
    expiresAt: Date
    existingPlayerIds: string[]
  },
  input: { now: Date; playerId: string | null },
) {
  if (room.expiresAt.getTime() <= input.now.getTime()) return { allowed: false, reason: "expired" }
  if (room.status === "REVIEW" || room.status === "ENDED") {
    const reviewOpen = room.reviewExpiresAt ? room.reviewExpiresAt.getTime() > input.now.getTime() : false
    return { allowed: reviewOpen && Boolean(input.playerId && room.existingPlayerIds.includes(input.playerId)), reason: "review" }
  }
  return { allowed: true, reason: "open" }
}

export function runInstantRunoffElection(input: {
  candidateIds: string[]
  ballots: string[][]
}): {
  winnerId: string | null
  rounds: Array<{ tally: Record<string, number>; eliminatedCandidateIds: string[] }>
}
```

Ranked-choice tie behavior: when no majority exists, eliminate the candidate or candidates with the fewest votes. If eliminating all remaining candidates would happen, choose the alphabetically first remaining candidate as the winner to keep the rule deterministic.
